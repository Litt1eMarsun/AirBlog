motivation

1. 前Mem0提供的记忆检索主要是通过**向量相似度检索**。

在对Mem0做评测 [Mem0论文学习、Demo、Benchmark复现](https://infinigence.feishu.cn/wiki/IWAkwNocuiHo36kBcUCcHkUenXd) 时，对评测结果进行分析，发现有以下特点：

+ 逻辑简单、答案比较确定的问题（例如某件事情何时发生、某人喜欢什么食物）容易回答正确
+ 比较抽象、需要逻辑推理的问题（例如问两个人的共同点）不容易回答正确
+ 相关度高的记忆不一定是跟答案有关系的。检索时需要多检索一些相关记忆，再让大模型生成回答。



使用：

（目前理解的是将mem0作为一个操作记忆数据库的SDK，LLM来调用mem0）

1. agent自动调用 agent-framework的api作为工具，对记忆进行增删改查
2. 在记忆平台里
+ 增删改
    - 是通过将召回的相关记忆和当前对话中的相关记忆+各个api写一个prompt作为输入
    - 给到llm，让LLM调用mem0的sdk，去操作数据库当中的记忆	
+ 查
    - 直接通过让Mem0 embedding查询数据库中的知识
3. add方法自动添加有用记忆（用户画像之类的），无用的不添加
4. 自动维护记忆，如果先喜欢后不喜欢，会删掉一开始的记忆
