http过程

多层，数据链路层，网络层，传输层，应用层

| 层级 | 举例 | 作用 |
| --- | --- | --- |
| 应用层 | HTTP, HTTPS, FTP | 定义“数据的语义”——比如发网页请求 |
| 传输层 | TCP, UDP | 定义“数据如何可靠地传送” |
| 网络层 | IP | 定义“数据包从哪来、送到哪去” |
| 数据链路层 | 以太网、Wi-Fi | 定义“在局域网内如何传输电信号” |


2. 应用层：
    1. 关系通信传送了什么内容，比如发送什么请求（请求状态）通过具体的服务器路径（路径）中获得信息即可，比如下述

```python
  conn.request("POST", "/search", payload, headers)
```

3. 传输层确保了网络通信能够正常进行，确保安全可靠
    1. TCP协议：

TCP的过程就是三次握手的过程，三次握手的具体过程如下：

> 客户端 → SYN → 服务器
>
> 服务器 → SYN+ACK → 客户端
>
> 客户端 → ACK → 服务器
>

b. 和Http之间的关系：

| HTTP 版本 | 是否复用 TCP | 机制 |
| --- | --- | --- |
| HTTP/1.0 | ❌ 否 | 每个请求都要新建连接 |
| HTTP/1.1 | ✅ 是 | 默认启用 `Connection: keep-alive` |
| HTTP/2 | ✅✅ 更高效 | 同一个 TCP 连接上可同时传多个请求（多路复用） |


## CDP协议
CDP是建立在浏览器内核创建的调试服务器和CDP客户端之间的websocket上的通信协议

websocket是基于TCP去做的，持久型的传输层，可以用于任何TCP上

HTTP是两个服务器之间的网页的应用层上的通信协议
